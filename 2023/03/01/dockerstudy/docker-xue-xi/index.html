docker学习

最近想要使用github上的一些关于wechat的以及chatgpt的机器人仓库，但是发现都需要使用docker，所以就开始学习docker，这里记录一下学习过程

比较Docker和虚拟机技术的不同

  1. 架构和性能：虚拟机技术模拟整个操作系统，包括硬件和内核，因此需要更多的资源，启动和关闭速度慢，性能略低。而Docker使用操作系统的内核，可以共享主机系统的资源，启动和关闭速度快，性能高。
  2. 大小和轻量级：虚拟机技术需要运行完整的操作系统，因此镜像文件通常比较大，占用更多的磁盘空间。而Docker使用容器技术，容器内只包含应用程序和依赖项，因此容器文件相对较小，占用更少的磁盘空间。
  3. 隔离级别：虚拟机技术提供硬件级别的隔离，每个虚拟机都有自己的磁盘、网络接口和其他硬件资源。而Docker使用操作系统级别的隔离，容器共享主机系统的硬件资源，但每个容器有自己的文件系统、网络接口和进程空间。
  4. 部署和管理：虚拟机技术需要一个完整的操作系统来运行，因此它们的部署和管理比较复杂。而Docker容器只需要应用程序和依赖项，因此它们可以更容易地部署和管理，可以通过Dockerfile和Docker Compose来定义和管理容器。
  5. 应用场景：虚拟机技术通常用于隔离不同的操作系统或在不同的硬件平台上运行应用程序。而Docker容器更适合于在同一台机器上运行多个隔离的应用程序，例如微服务架构、持续集成和部署等场景。
  6. 传统虚拟机，虚拟出一条硬件，运行一个完整的操作系统，然后在这个系统上安装和运行软件
  7. 容器内的应用直接运行在宿主机的内容，容器是没有自己的内核的，也没有虚拟我们的硬件，所以就轻便了
  8. 每个容器间是互相隔离，每个容器内都有一个属于自己的文件系统，互不影响。

总之，虚拟机技术提供更高的隔离级别，但需要更多的资源和管理复杂度，而Docker容器提供更轻量级、更快速的部署和管理方式，适用于在同一台机器上运行多个应用程序的场景。

一些常用命令的含义

docker run

  1. 在docker run命令中，-d选项表示将Docker容器作为后台（即守护进程）运行。-d选项是--detach选项的简写形式。（detach：使分离；脱离，摆脱；派遣，分派）

  2. 在docker run命令中，-t选项表示要为容器分配一个伪终端（pseudo-tty），以便可以在容器中运行交互式命令行应用程序。"Pseudo-tty"是伪终端的简称，其中“tty”是终端（teletype）的缩写，"Pseudo"是一个希腊词汇，意思是“假的”，或者“虚假的”。

docker ps

  1. docker ps命令用于列出当前正在运行的Docker容器。ps是“process status”的缩写，因此，docker ps可以被理解为“列出Docker容器的状态”。

对于下列命令的解释

这是一个Docker命令，它会运行一个容器。以下是各个参数的含义：

  • docker run：启动一个新的容器。
  • -e TZ="Asia/Shanghai"：设置容器的环境变量TZ为Asia/Shanghai，表示容器运行的时区是中国上海。
  • --name=chatBot：为容器指定名称chatBot。
  • --volume="$(pwd)/config/":/bot/config：将当前工作目录下的config目录挂载到容器内的/bot/config目录中。这个选项使用了命令替换语法$(pwd)，它会被当前工作目录所替换（所以要在当前目录新建配置文件config/index.js，并且将其中的参数配置修改成自己的）。这个选项的作用是将本地的配置文件与容器内的应用程序进行关联。
  • aibotk/wechat-bot：指定要启动的Docker镜像的名称，这个镜像名为aibotk/wechat-bot，表示使用的是一个名为wechat-bot的微信机器人应用程序镜像。

综上所述，这个Docker命令会创建一个名为chatBot的容器，运行一个名为wechat-bot的微信机器人应用程序，这个应用程序会使用本地文件系统中的配置文件，并且容器的时区设置为中国上海
