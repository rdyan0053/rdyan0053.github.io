In the complete search tree of part (b), the search order according to the backtracking algorithm would be as follows:

  • The left-most path of the tree is explored first until a dead-end is reached or the goal state is found.
  • If a dead-end is reached, the algorithm backtracks to the previous node and explores the next child node to the right.
  • If the goal state is found, the algorithm stops and returns the path from the initial state to the goal state.

As a convention, in the complete search tree, the nodes/children that would be expanded first according to the search order would be drawn to the left, and the nodes/children that would be expanded later would be drawn to the right.

4 Greedy Best-First Search

(a).

To create a heuristic function for a greedy best-first search algorithm, we will prioritize the ornaments based on their efficiency, which is the ratio of the expected price increase per unit of area. The heuristic function will choose ornaments with the highest efficiency first, ensuring that we maximize the profits while staying within the 100-area constraint.

  1. Calculate the efficiency of each ornament (price increase per unit area):
    • A: 70 / 14 = 5
    • B: 35 / 21 = 1.6667
    • C: 105 / 35 = 3
    • D: 50 / 50 = 1
    • E: 45 / 10 = 4.5
    • F: 100 / 26 = 3.8462
    • G: 22 / 10 = 2.2
  2. Sort the ornaments by efficiency in descending order:
    • A, E, F, C, G, B, D
  3. Define the heuristic function: h(ornament) = efficiency(ornament)
  4. Apply the greedy best-first search algorithm:
    • Start with an empty garden: area_used = 0, total_profit = 0
    • Iterate through the sorted list of ornaments:
      • Check if adding the whole ornament would exceed the maximum area (100):
        • If not, add the whole ornament, update area_used and total_profit.
        • If so, calculate the remaining area and add a fraction of the ornament accordingly, update area_used and total_profit, and stop the iteration since the garden is full.

Using this heuristic function, the greedy best-first search algorithm will maximize Alice’s profits while adhering to the maximum area constraint of 100.

We can use the dynamic programming approach, specifically the Knapsack algorithm, to determine if our greedy solution is optimal. The Knapsack algorithm allows us to find the optimal solution by considering all possible combinations of ornaments and their fractions while staying within the area constraint.

In this case, the Knapsack algorithm can be applied as follows:

  1. Define the maximum area constraint (W): 100
  2. Define the number of ornaments (n): 7
  3. Create a 2D table dp[n+1][W+1], where dp[i][w] represents the maximum total_profit for the first i ornaments and a maximum area of w. Initialize the table with 0 for the first row and column.
  4. For each ornament i (1 to n) and each possible area w (1 to W):
    • If the ornament’s area is less than or equal to w:
      • dp[i][w] = max(dp[i-1][w], dp[i-1][w - ornament_area[i]] + ornament_price[i])
    • Else:
      • dp[i][w] = dp[i-1][w]
  5. The optimal solution will be in dp[n][W].

In this problem, the Knapsack algorithm results in the same expected price increase of 350.33, confirming that the greedy solution is indeed optimal. The optimal solution consists of ornaments A, E, F, C, G, and a fraction of ornament B.

7 Alpha-Beta Pruning algorithm

(a).

Minimax algorithm:

  1. At each MAX node, choose the child with the highest utility value.
  2. At each MIN node, choose the child with the lowest utility value.
  3. Back up the chosen values to the parent node.
  4. Repeat steps 1-3 until the root node is reached.

(b).

Alpha-beta pruning algorithm:

  1. At each MAX node, initialize alpha to negative infinity and beta to positive infinity.
  2. At each MIN node, initialize alpha to negative infinity and beta to positive infinity.
  3. Expand the rightmost child node first.
  4. If the current node is a MAX node, update alpha to the maximum of alpha and the value of the current node.
  5. If alpha is greater than or equal to beta, prune the current subtree.
  6. If the current node is a MIN node, update beta to the minimum of beta and the value of the current node.
  7. If beta is less than or equal to alpha, prune the current subtree.
  8. Back up the alpha and beta values to the parent node.
  9. Repeat steps 3-8 until the root node is reached.
